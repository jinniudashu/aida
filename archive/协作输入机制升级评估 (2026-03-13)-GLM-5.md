# 协作输入机制升级评估

**评估日期：2026-03-13**
**提案人：用户**
**评估模型：GLM-5**

---

## 一、背景与问题陈述

### 1.1 核心洞察

**"人类输入"应抽象为"外部协作输入"**——我们无法确定完成这个协作任务的是人类还是另一个 Agent。这种抽象使得 AIDA 可以支持：

- **HITL（Human-in-the-Loop）**：人类操作者提供输入
- **AITL（Agent-in-the-Loop）**：外部 Agent（如专家系统、审批服务）提供输入
- **混合场景**：人类 + AI 协作完成

### 1.2 当前实现

AIDA 的协作输入机制目前仅支持**简单审批**：

| 组件 | 实现细节 |
|------|----------|
| 数据模型 | `ApprovalRequest`（id, tool, toolInput, status: PENDING/APPROVED/REJECTED/TIMEOUT） |
| 存储 | `bps_management_approvals` 表 |
| API | `POST /api/management/approvals/:id/decide` → `{decision: APPROVED/REJECTED}` |
| 前端 | ManagementPage.vue → Approve/Reject 按钮 |
| 触发 | 管理层约束 `onViolation: REQUIRE_APPROVAL` |

**核心限制**：协作方只能提供 **Yes/No** 二元信息。

### 1.3 用户建议

> "审批"是协作输入的一种最简单方式。实际场景中，协作方提供的信息类型还有：
> - **结构化输入**（表单）
> - **自然语言文本**
> - 多媒体（暂不考虑）
>
> 建议升级 Dashboard "审批"页面为"待办"页面，作为所有协作任务的链接列表入口。

## 二、HITL 信息类型分析

### 2.1 信息类型光谱

```
低信息量                                    高信息量
   |                                           |
   v                                           v
[Yes/No] ← [选择] ← [表单] ← [文本] ← [多媒体]
   |          |         |        |         |
 审批       分类     结构化    自由文本   附件
```

| 类型 | 信息量 | 典型场景 | AIDA 适用性 |
|------|--------|----------|-------------|
| Yes/No | 1 bit | 审批、确认 | **当前支持** |
| 选择（枚举） | log₂(n) bits | 分类、优先级 | 高 |
| 表单（结构化） | 可变 | 参数补充、配置 | **高** |
| 文本 | 无限 | 澄清、指导、反馈 | **中-高** |
| 多媒体 | 大 | 图片标注、语音 | 低（暂不考虑） |

### 2.2 AIDA 场景映射

| 场景 | 当前支持 | 协作方需要提供的信息 | 建议类型 |
|------|---------|-------------------|----------|
| 敏感操作审批 | ✓ | Yes/No | 保持 |
| 参数补充 | ✗ | 缺失字段值（如预算上限） | **表单** |
| 内容确认/修正 | ✗ | 文本修改、确认 | **文本** |
| 歧义澄清 | ✗ | 选择正确的实体/意图 | **选择** |
| 优先级排序 | ✗ | 任务优先级 | **选择** |
| 策略指导 | ✗ | 高层指示 | **文本** |
| 异常处理 | ✗ | 处理方式选择 | **选择/表单** |

### 2.3 协作方身份抽象

协作方（Collaborator）可以是：

| 身份 | 特征 | 触发方式 |
|------|------|----------|
| **人类操作者** | 通过 Dashboard/飞书/邮件响应 | Dashboard 待办页 |
| **外部 Agent** | 通过 API/Webhook 响应 | `POST /api/collaboration/respond` |
| **系统服务** | 自动化规则引擎 | 内部回调 |

**关键设计**：任务创建时不指定具体协作方身份，只声明所需输入类型和 Schema。响应时由协作方自声明身份。

## 三、待办机制设计

### 3.1 核心概念模型

```
┌─────────────────────────────────────────────────────────────┐
│                    CollaborationTask                         │
│  (协作任务 - 通用待办项)                                      │
├─────────────────────────────────────────────────────────────┤
│ id: string                                                   │
│ type: 'approval' | 'form' | 'choice' | 'text' | 'mixed'     │
│ title: string                                                │
│ description: string                                          │
│ context: { agent, tool, entity, ... }                        │
│ inputSchema: JSONSchema (定义期望的输入结构)                  │
│ response: { data, respondedBy, respondedAt } | null          │
│ status: 'pending' | 'completed' | 'expired' | 'cancelled'    │
│ priority: 'low' | 'normal' | 'high' | 'urgent'               │
│ createdAt, expiresAt, completedAt                            │
│ createdBy: { type: 'agent' | 'system', id: string }          │
└─────────────────────────────────────────────────────────────┘
```

**关键设计**：
- `createdBy` 不再是字符串，而是 `{type, id}` 结构，明确是哪个 Agent/系统创建的
- `response` 包含 `respondedBy: { type: 'human' | 'agent' | 'system', id: string }`，由响应方自声明身份
- 任务创建时**不指定**具体协作方，任何有能力响应的实体都可以响应

### 3.2 类型定义

```typescript
// src/collaboration/types.ts

export type CollaborationTaskType =
  | 'approval'      // Yes/No
  | 'choice'        // 单选/多选
  | 'form'          // 结构化表单
  | 'text'          // 自由文本
  | 'mixed'         // 混合类型
  ;

export type CollaborationTaskStatus =
  | 'pending'
  | 'completed'
  | 'expired'
  | 'cancelled'
  ;

export type CollaborationTaskPriority = 'low' | 'normal' | 'high' | 'urgent';

// 协作方身份
export interface CollaboratorIdentity {
  type: 'human' | 'agent' | 'system';
  id: string;  // human: user-id, agent: agent-id, system: service-name
  label?: string;  // 可读名称
}

export interface CollaborationTask {
  id: string;
  type: CollaborationTaskType;

  // 展示信息
  title: string;
  description: string;

  // 上下文（用于 Dashboard 展示和决策参考）
  context: {
    agentId?: string;
    tool?: string;
    toolInput?: Record<string, unknown>;
    entityType?: string;
    entityId?: string;
    sessionId?: string;
    metadata?: Record<string, unknown>;
  };

  // 输入 Schema（定义期望的输入结构）
  // approval: { type: 'boolean' }
  // form: { type: 'object', properties: {...} }
  // choice: { type: 'string', enum: ['A', 'B', 'C'] }
  // text: { type: 'string' }
  inputSchema: Record<string, unknown>;

  // 协作方响应
  response: {
    data: Record<string, unknown>;
    respondedBy: CollaboratorIdentity;
    respondedAt: string;
  } | null;

  // 状态与优先级
  status: CollaborationTaskStatus;
  priority: CollaborationTaskPriority;

  // 时间戳
  createdAt: string;
  expiresAt: string;
  completedAt?: string;

  // 来源（创建方）
  createdBy: CollaboratorIdentity;
}

// Agent 侧请求接口
export interface CreateCollaborationTaskRequest {
  type: CollaborationTaskType;
  title: string;
  description: string;
  context?: CollaborationTask['context'];
  inputSchema?: Record<string, unknown>;  // 可选，有默认值
  priority?: CollaborationTaskPriority;
  expiresIn?: string;  // 如 '24h', '7d'
}

// 协作方响应接口
export interface SubmitCollaborationResponse {
  taskId: string;
  response: Record<string, unknown>;
  respondedBy: CollaboratorIdentity;
}
```

### 4.1 新增 BPS Tool：`bps_request_collaboration`

```typescript
// src/integration/tools.ts

const RequestCollaborationSchema = Type.Object({
  type: Type.Union([
    Type.Literal('approval'),
    Type.Literal('choice'),
    Type.Literal('form'),
    Type.Literal('text'),
    Type.Literal('mixed'),
  ], { description: 'Type of collaboration input required' }),

  title: Type.String({ description: 'Short title for the task' }),
  description: Type.String({ description: 'Detailed description of what is needed' }),

  context: Type.Optional(Type.Object({
    entityType: Type.Optional(Type.String()),
    entityId: Type.Optional(Type.String()),
    metadata: Type.Optional(Type.Record(Type.String(), Type.Unknown())),
  })),

  inputSchema: Type.Optional(Type.Record(Type.String(), Type.Unknown(), {
    description: 'JSON Schema defining expected input. Auto-generated for approval/choice types.'
  })),

  options: Type.Optional(Type.Array(Type.String(), {
    description: 'For choice type: list of options to choose from'
  })),

  priority: Type.Optional(Type.Union([
    Type.Literal('low'),
    Type.Literal('normal'),
    Type.Literal('high'),
    Type.Literal('urgent'),
  ])),

  expiresIn: Type.Optional(Type.String({
    description: 'Expiration time (e.g., "24h", "7d"). Default: 24h'
  })),
});

function createRequestCollaborationTool(deps: BpsToolDeps): OpenClawAgentTool {
  return {
    name: 'bps_request_collaboration',
    description: `Request external collaboration input (HITL or AITL).

Types:
- approval: Yes/No decision (default schema provided)
- choice: Select from predefined options
- form: Structured input (provide custom schema)
- text: Free-form text response
- mixed: Combination of above

The task appears in Dashboard "待办" page and can be responded by:
1. Human operator (via Dashboard)
2. External Agent (via API)
3. System service (via internal callback)

Use when:
1. You need a decision before proceeding (approval)
2. You need to clarify ambiguous input (choice)
3. You need additional structured data (form)
4. You need guidance or feedback (text)

Returns: taskId (check status via bps_get_collaboration_response)`,
    parameters: RequestCollaborationSchema,
    async execute(_callId: string, input: unknown) {
      const params = input as CreateCollaborationTaskRequest & { options?: string[] };

      // Auto-generate schema for approval/choice
      let schema = params.inputSchema;
      if (!schema) {
        if (params.type === 'approval') {
          schema = DEFAULT_SCHEMAS.approval;
        } else if (params.type === 'choice' && params.options) {
          schema = {
            type: 'object',
            properties: {
              selected: { type: 'string', enum: params.options },
              reason: { type: 'string' },
            },
            required: ['selected'],
          };
        } else if (params.type === 'text') {
          schema = DEFAULT_SCHEMAS.text;
        }
      }

      const task = deps.collaborationStore.createTask({
        type: params.type,
        title: params.title,
        description: params.description,
        context: params.context ?? {},
        inputSchema: schema ?? {},
        priority: params.priority ?? 'normal',
        expiresIn: params.expiresIn,
      });

      return {
        success: true,
        taskId: task.id,
        status: task.status,
        dashboardUrl: `/inbox/${task.id}`,
        hint: `Task created. Collaborator can respond at Dashboard "待办" page or via API. Check status with bps_get_collaboration_response(taskId="${task.id}")`,
      };
    },
  };
}
```

### 4.2 新增 BPS Tool：`bps_get_collaboration_response`

```typescript
function createGetCollaborationResponseTool(deps: BpsToolDeps): OpenClawAgentTool {
  return {
    name: 'bps_get_collaboration_response',
    description: 'Check if a collaborator has responded to a previously requested task.',
    parameters: Type.Object({
      taskId: Type.String({ description: 'The task ID from bps_request_collaboration' }),
      wait: Type.Optional(Type.Boolean({ description: 'If true, wait until response (max 5min). Default: false' })),
    }),
    async execute(_callId: string, input: unknown) {
      const { taskId, wait } = input as { taskId: string; wait?: boolean };

      const task = deps.collaborationStore.getTask(taskId);
      if (!task) {
        return { error: `Task not found: ${taskId}` };
      }

      if (wait && task.status === 'pending') {
        // Poll until response or timeout (simplified)
        const startTime = Date.now();
        const timeout = 5 * 60 * 1000; // 5 minutes
        while (Date.now() - startTime < timeout) {
          await new Promise(r => setTimeout(r, 2000));
          const updated = deps.collaborationStore.getTask(taskId);
          if (updated && updated.status !== 'pending') {
            return {
              taskId: updated.id,
              status: updated.status,
              response: updated.response?.data,
              respondedBy: updated.response?.respondedBy,
              respondedAt: updated.response?.respondedAt,
            };
          }
        }
        return { taskId, status: 'pending', hint: 'Timeout waiting for response' };
      }

      return {
        taskId: task.id,
        status: task.status,
        response: task.response?.data,
        respondedBy: task.response?.respondedBy,
        respondedAt: task.response?.respondedAt,
      };
    },
  };
}
```

### 6.1 数据库 Schema

```sql
-- 通用协作任务表（替代 bps_management_approvals）
CREATE TABLE IF NOT EXISTS bps_collaboration_tasks (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,              -- approval/choice/form/text/mixed
  title TEXT NOT NULL,
  description TEXT NOT NULL,

  -- 上下文
  context TEXT NOT NULL DEFAULT '{}',

  -- 输入 Schema
  input_schema TEXT NOT NULL DEFAULT '{}',

  -- 响应

  responded_by_type TEXT,           -- human/agent/system
  responded_by_id TEXT,              -- 具体ID
  responded_at TEXT,

  status TEXT NOT NULL DEFAULT 'pending',
  priority TEXT NOT NULL DEFAULT 'normal',

  -- 时间戳
  created_at TEXT NOT NULL,
  expires_at TEXT NOT NULL,
  completed_at TEXT,

  -- 来源（创建方）
  created_by_type TEXT NOT NULL,    -- agent/system
  created_by_id TEXT NOT NULL,       -- 具体id

  -- 索引优化
  entity_type TEXT,
  entity_id TEXT
);

CREATE INDEX IF NOT EXISTS idx_collaboration_tasks_status ON bps_collaboration_tasks(status, priority, created_at);
CREATE INDEX IF NOT EXISTS idx_collaboration_tasks_type ON bps_collaboration_tasks(type, status);
CREATE INDEX IF NOT EXISTS idx_collaboration_tasks_expires ON bps_collaboration_tasks(expires_at) WHERE status = 'pending';
```

### 6.2 迁移策略（直接替换）

由于尚未进入业务测试，直接替换现有审批机制：

| 组件 | 旧 | 新 |
|------|-----|-----|
| 数据库表 | `bps_management_approvals` | `bps_collaboration_tasks` |
| API 路径 | `/api/management/approvals` | `/api/collaboration/tasks` |
| 前端页面 | `ManagementPage.vue`（审批标签） | `InboxPage.vue`（待办页面） |
| BPS Tool | `bps_management_status` | `bps_collaboration_status` |

**迁移脚本**：
```sql
-- 一次性迁移
INSERT INTO bps_collaboration_tasks
  (id, type, title, description, context, input_schema, response, status, priority, created_at, expires_at, created_by_type, created_by_id)
SELECT
  id,
  'approval' as type,
  '审批请求' as title,
  message as description,
  json_object('tool', tool, 'toolInput', tool_input) as context,
  '{"type":"object","properties":{"decision":{"type":"boolean"},"reason":{"type":"string"}},"required":["decision"]}' as input_schema,
  CASE
    WHEN status = 'APPROVED' THEN json_object('decision', true, 'approvedBy', approved_by, 'respondedAt', decided_at)
    WHEN status = 'REJECTED' THEN json_object('decision', false, 'approvedBy', approved_by, 'respondedAt', decided_at)
    ELSE null
  END as response,
  LOWER(status) as status,
  'normal' as priority,
  created_at,
  expires_at,
  'agent' as created_by_type,
  'bps-engine' as created_by_id
FROM bps_management_approvals;

-- 删除旧表
DROP TABLE bps_management_approvals;
```

## 七、与 OpenClaw 的集成

### 7.1 OpenClaw 原生审批

OpenClaw 有 `exec.approval` 机制，但它是命令级审批，不支持结构化输入。

**结论**：AIDA 协作输入是业务层概念，与 OpenClaw approval 互补。

### 7.2 Steer 机制

当协作方响应待办时，可以通过 OpenClaw 的 `steer` 机制注入指令到正在运行的 Agent Session：

```typescript
// 协作方提交响应后
api.emit('steer', {
  sessionKey: task.context.sessionId,
  message: `Collaborator responded: ${JSON.stringify(task.response)}`,
});
```

## 十、结论

### 核心收益

1. **身份抽象**：从"人类输入"升级为"外部协作输入"，支持 HITL 和 AITL
2. **信息量提升**：从 1 bit（Yes/No）到无限制（表单+文本）
3. **场景覆盖**：从单一审批扩展到参数补充、歧义澄清、策略指导
4. **统一入口**：所有协作任务集中在"待办"页面

### 建议行动

1. **P0 立即**：实现 `CollaborationTask` 基础设施 + `approval` 类型
2. **P1 短期**：实现 `form` + `text` 类型
3. **P2 中期**：实现 `choice` + `mixed` + SSE 实时推送

### 架构原则

- **身份抽象**：协作方可以是人类、也可以是外部 Agent
- **Schema 驱动**：前端渲染从 Schema 自动生成，后端可扩展
- **类型安全**：TypeScript 类型覆盖全链路
- **直接替换**：由于尚未进入业务测试，直接替换旧机制
